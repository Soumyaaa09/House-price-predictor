# 🏠 EstateIQ — House Price Prediction System

A complete ML + Flask web application for predicting house prices.

---

## 📂 Project Structure

```
house_price_prediction/
├── app.py                  ← Flask backend
├── model.py                ← ML training script
├── house_model.pkl         ← Saved model (generated by model.py)
├── requirements.txt        ← Python dependencies
├── dataset/
│   └── housing.csv         ← Training dataset (generated by model.py)
├── templates/
│   ├── index.html          ← Home page (prediction form)
│   └── result.html         ← Results page
└── static/
    ├── css/style.css       ← Stylesheet
    └── js/script.js        ← JavaScript interactions
```

---

## ▶️ How to Run (Step-by-Step)

### Step 1 — Open the project in VS Code
```
File → Open Folder → select house_price_prediction/
```

### Step 2 — Create & activate a virtual environment
```bash
python -m venv venv

# Windows:
venv\Scripts\activate

# macOS/Linux:
source venv/bin/activate
```

### Step 3 — Install dependencies
```bash
pip install -r requirements.txt
```

### Step 4 — Train the ML model
```bash
python model.py
```
This generates `dataset/housing.csv` and `house_model.pkl`.

### Step 5 — Start the Flask server
```bash
python app.py
```

### Step 6 — Open in browser
```
http://127.0.0.1:5000
```

## 🧠 ML Details

| Property          | Value                       |
|-------------------|-----------------------------|
| Algorithm         | Gradient Boosting Regressor |
| Training records  | 1,000                       |
| Features          | Area, Bedrooms, Bathrooms, Location, Age, Garage |
| Train/Test split  | 80% / 20%                   |
| Typical R² Score  | ~0.98                       |
| Evaluation        | R², MAE, RMSE               |

## ⚠️ Notes

- Run `model.py` before `app.py` — the app won't start without `house_model.pkl`
- Bathrooms cannot exceed bedrooms (validated on both frontend and backend)
- Dataset is synthetically generated but follows realistic price distributions
